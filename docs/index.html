The Problem

Claude has a reading problem.

Every exploration task starts with an avalanche of file reads. Large files get pulled in whole. The same file gets read 3, 4, 5 times in a single session. Each read eats context tokens that could be used for actual thinking.

0

Wasted Reads

Average redundant file reads in a complex coding session. Each one consumes tokens for content Claude already saw.

40%

Context Bloat

Estimated context window consumed by re-reading files that were already loaded earlier in the session.

100KB

Blind Reads

Claude reads entire files without knowing what's inside. A 2000-line file gets loaded just to check one function signature.

∞

No Memory

Each read has zero awareness of prior reads. No built-in "hey, you already read this" mechanism exists.

How It Works

A quiet interceptor.

youwhatknow runs as a system-wide daemon and hooks into Claude Code's tool lifecycle. Before every file read, it injects a compact summary. Claude can then decide if it actually needs the full file.

1

Claude triggers a Read

Claude Code fires a PreToolUse hook before executing the Read tool. The hook sends the file path and session ID to youwhatknow via HTTP.

POST /hook/pre-read
2

Summary injected

youwhatknow looks up the file in its index and returns a 3-line summary: description, public symbols, and folder context. This gets injected into Claude's context before the read happens.

additionalContext: "Public: Config, load(), default()"
3

Re-read warning

If Claude already read this file in the current session, youwhatknow flags it: "This file was already read 3x. Do you still need the full file?"

read 3x this session
4

Project map on startup

On SessionStart, the full project structure is injected so Claude knows where everything is without exploring.

POST /hook/session-start
Architecture

One daemon, all projects.

A single system-wide process serves every Claude Code instance across all your projects. Indexes are loaded lazily per-project on first request.

Benefits

What you actually get.

</>

Smaller Context

Summaries are 3 lines instead of 2000. Claude gets what it needs without filling the window.

↺

Re-read Awareness

Per-session tracking warns Claude when it's reading a file again. Breaks the compulsive loop.

∅

Zero Config

One HTTP hook in settings.json. The daemon handles everything: indexing, session tracking, multi-project routing.

▦

Multi-Project

Single daemon serves all projects. Indexes loaded lazily on first request. No per-project setup.

↻

Incremental Index

Only re-indexes files that changed since last run. Uses git diff for fast change detection.

✕

Non-Blocking

If the daemon isn't running, Claude works normally. HTTP hooks fail silently. No disruption, ever.

Get Started

Two steps. Done.

~/.claude/settings.json step 1: hook config
{ "hooks": { "PreToolUse": [{ "matcher": "Read", "hooks": [{ "type": "http", "url": "http://localhost:7849/hook/pre-read" }] }], "SessionStart": [{ "matcher": "startup", "hooks": [{ "type": "http", "url": "http://localhost:7849/hook/session-start" }] }] } }
terminal step 2: start the daemon
# Install and run (or add to your shell profile / nix flake) $ youwhatknow # That's it. Indexes projects lazily on first hook request. # Auto-shuts down after 30 min idle. Restarts on next Claude session.
~/.config/youwhatknow/config.toml optional
# All settings have sensible defaults port = 7849 session_timeout_minutes = 60 idle_shutdown_minutes = 30
