How it works

A polite but firm intervention.

youwhatknow sits between Claude and the filesystem like a responsible friend at a buffet. "You can have more if you want, but here's what's on your plate already."

1

Claude reaches for a file

Claude Code fires a PreToolUse hook before every Read. youwhatknow gets the file path and session ID via HTTP before the read happens.

POST /hook/pre-read
2

A 3-line summary appears

Instead of 2000 lines of code, Claude sees: file description, public symbols, and folder context. Three lines. That's usually enough.

"Entry point, server startup. Public: main()"
Think of it as the back-cover blurb instead of reading the whole book.
3

Repeat offenders get called out

Second read? Gentle note. Third read? "This file was already read 3x this session. Summary below — do you still need the full file?"

read 3x this session
Tough love. But the kind that saves your token budget.
4

Day-one orientation

On SessionStart, Claude gets a full project map injected automatically. No more "let me explore the codebase" spirals. It already knows where everything is.

POST /hook/session-start
Like giving the new hire a map instead of letting them wander the building.
Architecture

One daemon to rule them all.

A single process serves every Claude Code session across all your projects. Spin up 5 terminals with 5 projects? Same daemon. Subagents? Same daemon. It figures out which project you're in from the cwd in each request.

The good stuff

What's in it for you.

</>

Context on a diet

3 lines instead of 2000. Claude gets the gist without the bloat. More room for actual thinking.

↺

"You already read that"

Per-session tracking breaks the compulsive re-read loop. A gentle nudge before Claude goes for round 5.

∅

Stupid simple setup

Paste a hook config. Start the daemon. That's it. No per-project ceremony. No YAML nightmares.

▦

All projects, one process

The daemon loads project indexes lazily. First request for a new project? Indexed in the background. Zero waiting.

↻

Keeps itself fresh

Incremental re-indexing via git diff. Only changed files get re-processed. Fast and quiet.

✕

Invisible when off

Daemon not running? Claude works normally. HTTP hooks fail silently. It's there when you want it, gone when you don't.

Get started

Two steps. Seriously.

~/.claude/settings.json step 1: tell claude about youwhatknow
{ "hooks": { "PreToolUse": [{ "matcher": "Read", "hooks": [{ "type": "http", "url": "http://localhost:7849/hook/pre-read" }] }], "SessionStart": [{ "matcher": "startup", "hooks": [{ "type": "http", "url": "http://localhost:7849/hook/session-start" }] }] } }
terminal step 2: let it rip
# Start the daemon. It indexes your projects lazily. $ youwhatknow # It shuts itself down after 30 min of no action. # Next time Claude starts a session, the shell hook brings it back. # Very chill. Very low maintenance. Like a good houseplant.
~/.config/youwhatknow/config.toml optional — for the tinkerers
# All settings have sensible defaults. You probably don't need this file. port = 7849 # daemon port session_timeout_minutes = 60 # stale session cleanup idle_shutdown_minutes = 30 # auto-shutdown when bored
